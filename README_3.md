# Домашнее задание по 1С  
## Тема: Учет фонда оплаты труда по сотрудникам

### Описание работы
В рамках данного задания в системе **1С:Предприятие** был реализован механизм накопления и хранения информации о суммах выплат заработной платы по каждому сотруднику организации за всё время его работы.  
Для решения задачи использован **регистр накопления** и документ **«Начисление зарплаты»**, который формирует движения по этому регистру при проведении.

### Цель работы
Автоматизировать процесс учета общего объема выплат заработной платы по каждому сотруднику организации и обеспечить возможность накопления этих данных в системе 1С.

### Постановка задачи
Необходимо было реализовать механизм, благодаря которому система будет сохранять сведения о том, сколько денежных средств получил каждый сотрудник за весь период работы.  
Согласно условию задания, при формировании движений регистра должен использоваться только вид движения **«Расход»**.

### Выполненные этапы
В ходе выполнения работы были реализованы следующие шаги:

1. **Создан регистр накопления `ФондОплатыТруда`**  
   Для хранения данных о выплатах заработной платы был добавлен новый регистр накопления с синонимом **«Фонд оплаты труда»**.

2. **Выбран вид регистра `Обороты`**  
   Такой тип регистра позволяет накапливать информацию о движениях денежных сумм по сотрудникам.

3. **Добавлено измерение `Сотрудник`**  
   В качестве измерения использован реквизит:
   - `Сотрудник`  
   Тип данных: ссылка на справочник **Сотрудники**.

4. **Добавлен ресурс `Сумма`**  
   В качестве ресурса регистра настроен реквизит:
   - `Сумма`  
   Тип данных: **Число**.

5. **Подключен регистр к документу `НачислениеЗарплаты`**  
   На вкладке **«Движения»** документа был добавлен регистр **`ФондОплатыТруда`**, что позволило документу формировать движения при проведении.

6. **Настроен конструктор движений регистров**  
   Для регистра был выбран:
   - тип движения: **Расход**;
   - источник данных: табличная часть **Сотрудники**.

7. **Заполнены выражения для полей регистра**
   В конструкторе движений были настроены следующие соответствия:
   - `Сотрудник = ТекСтрокаСотрудники.Сотрудник`
   - `Сумма = ТекСтрокаСотрудники.Сумма`

8. **Сформирован код проведения документа**
   В модуле объекта документа **`НачислениеЗарплаты`** была создана процедура **`ОбработкаПроведения`**, которая:
   - включает запись движений регистра;
   - проходит по всем строкам табличной части **Сотрудники**;
   - создает движение по регистру для каждой строки;
   - записывает сотрудника и сумму;
   - использует вид движения **Расход**.

9. **Проверена работа механизма в пользовательском режиме**
   После проведения документа в режиме **1С:Предприятие** были проверены записи регистра **«Фонд оплаты труда»**.  
   В результате в регистре отобразились движения по каждому сотруднику с корректными суммами выплат.

### Результат работы
В результате выполнения задания был реализован механизм накопления фонда оплаты труда по сотрудникам.  
Система теперь позволяет:
- сохранять сведения о суммах выплат каждому сотруднику;
- автоматически формировать движения при проведении документа;
- накапливать информацию о заработной плате сотрудников за всё время работы.

### Итог
Поставленная задача выполнена успешно.  
В конфигурации 1С создан регистр накопления **«ФондОплатыТруда»**, связанный с документом **«Начисление зарплаты»**. При проведении документа формируются движения вида **«Расход»**, благодаря чему в системе накапливаются данные о выплатах каждому сотруднику за весь период работы.
